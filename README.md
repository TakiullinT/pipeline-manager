# PipelineManager

REST API сервис на Java / Spring Boot для управления направленными графами пайплайнов с проверкой на циклы и вычислением топологического порядка выполнения.

## 📝 Описание проекта

Приложение предоставляет HTTP-интерфейс для конструирования пайплайнов. Пайплайн представляет собой структуру из узлов и направленных зависимостей между ними (без фактического выполнения самих задач).

**Ключевые особенности реализации:**
* **Богатая доменная модель (Rich Domain Model):** Вся бизнес-логика инкапсулирована внутри класса `Pipeline`, а сервисный слой выступает исключительно в роли оркестратора.
* **Защита от циклов (DAG):** При каждой попытке добавить новую зависимость используется алгоритм поиска в ширину. Если ребро создает цикл, транзакция отклоняется.
* **Топологическая сортировка:** Вычисление правильного порядка выполнения узлов реализовано на основе алгоритма Кана.
* **Детерминированность:** Использование `TreeSet` и `TreeMap` в доменной модели гарантирует лексикографическую сортировку узлов. Это делает ответы API и поведение тестов на 100% предсказуемыми от запуска к запуску.
* **Потокобезопасность:** В качестве In-memory хранилища применяется `ConcurrentHashMap`. Внутреннее состояние пайплайна защищено синхронизацией, а геттеры отдают неизменяемые снимки данных (`Set.copyOf`, `Map.copyOf`) для защиты от `ConcurrentModificationException`.
* **Надежность:** Валидация входных данных на уровне контроллеров (`@Valid`) и глобальная обработка исключений (`@ControllerAdvice`) с возвратом понятных HTTP-статусов (400, 404).

## 💾 Подход к хранению данных и архитектура
В текущей версии приложение использует легковесное In-Memory хранилище (`InMemoryPipelineRepository`), построенное на базе ConcurrentHashMap. Это решение выбрано умышленно для упрощения запуска, тестирования и демонстрации работы алгоритмов без необходимости разворачивать внешнюю инфраструктуру (`Docker`, `PostgreSQL` и т.д.).

Архитектурная готовность к масштабированию:

Dependency Inversion (`SOLID`): Сервисный слой зависит строго от абстрактного интерфейса PipelineRepository. Переход на персистентное хранилище (например, `Spring Data JPA` + `PostgreSQL`) потребует лишь добавления новой реализации репозитория, без единого изменения в бизнес-логике.

Потокобезопасность домена: При использовании In-Memory хранилища потокобезопасность обеспечивается на уровне самой доменной модели Pipeline. Все операции изменения графа синхронизированы, что предотвращает состояние гонки (`Race Conditions`) при параллельных HTTP-запросах.

## ⚙️ Требования к окружению

* **Java Development Kit (JDK) 22** (поддерживается 17 и выше)
* Установленный локально сборщик Maven/Gradle **не требуется** - проект содержит настроенный Wrapper (`mvnw` / `gradlew`).

## 🛠 Сборка и запуск

Проект использует Wrapper, поэтому ручная установка сборочной системы не требуется. Используйте терминал в корне проекта:

* **Сборка проекта:**
```bash
# Для Gradle
./gradlew build
```

* **Запуск:**

```bash
# Для Gradle
./gradlew bootRun
```

*Приложение будет запущено на порту `8080`.*

* **Запуск тестов:**

```bash
# Для Gradle
./gradlew test --info

```

## 📊 Примеры запросов (API)

Сервис принимает и возвращает данные в формате JSON. Ниже приведен базовый сценарий работы:

**1. Создание пайплайна:**

```bash
curl -X POST http://localhost:8080/pipelines

```

*Вывод: `{"id": "e12fb126...", "nodes": [], "edges": {}}`*

**2. Добавление узлов:**

```bash
curl -X POST http://localhost:8080/pipelines/e12fb126.../nodes \
  -H "Content-Type: application/json" \
  -d '{"nodeId": "input"}'

curl -X POST http://localhost:8080/pipelines/e12fb126.../nodes \
  -H "Content-Type: application/json" \
  -d '{"nodeId": "output"}'

```

**3. Добавление зависимостей (input -> output):**

```bash
curl -X POST http://localhost:8080/pipelines/e12fb126.../edges \
  -H "Content-Type: application/json" \
  -d '{"from": "input", "to": "output"}'

```

**4. Получение порядка выполнения (Execution Order):**

```bash
curl -X GET http://localhost:8080/pipelines/e12fb126.../execution-order

```

*Выходные данные (порядок узлов):*

```json
[
  "input",
  "output"
]

```
